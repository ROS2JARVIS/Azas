Azas 실로봇 연결부터 좌표 기록까지

교안 26~28장 기준: MoveIt 목표 Pose는 보통 base_link 기준이고, 좌표 기록은 base_link -> link_6 TF를 읽어서 수행한다.

핵심 개념

base_link

로봇이 생각하는 좌표 원점이다. 보통 로봇 베이스 중심을 기준으로 한다.

link_6

6번 관절 이후의 로봇 손목 끝 링크다. 그리퍼가 장착되는 기준 위치로 보면 된다.

TF 읽기

tf2_echo base_link link_6는 로봇 기준에서 손목 끝이 어디 있는지 보여준다.

핵심: 사람이 좌표를 상상해서 만드는 것이 아니라, 로봇 끝단을 원하는 위치로 가져간 뒤 그 순간의 TF 값을 기록한다.

안전 전제

  • 처음 연결은 자동 동작 목적이 아니라 좌표 확인 목적이다.
  • 비상정지 버튼과 티치펜던트를 바로 누를 수 있는 위치에서 진행한다.
  • RViz에서 임의로 Plan & Execute를 누르지 않는다.
  • Azas 작업에서는 /home/ssu/Azas 환경만 사용한다.
실측 전 임의 좌표를 실제 로봇 motion에 넣지 않는다. 좌표는 직접교시 또는 TF 확인으로 얻은 값만 사용한다.

1. 로봇 연결 준비

1

로봇 전원과 네트워크 확인

로봇 전원 ON, 비상정지 해제, 로봇 IP 확인. 예시는 192.168.1.100이다.

2

Azas 환경만 source

cd /home/ssu/Azas
source /opt/ros/humble/setup.bash
source /home/ssu/Azas/install/local_setup.bash
3

패키지 경로 확인

ros2 pkg prefix dsr_bringup2
ros2 pkg prefix dsr_moveit_config_m0609
ros2 pkg prefix moveit_py

정상이라면 대부분 /home/ssu/Azas/install/...로 잡힌다.

2. 실로봇 MoveIt bringup

터미널 1에서 실행한다. host는 실제 로봇 IP로 바꾼다.

cd /home/ssu/Azas
source /opt/ros/humble/setup.bash
source /home/ssu/Azas/install/local_setup.bash

ros2 launch dsr_bringup2 dsr_bringup2_moveit.launch.py \
  mode:=real \
  model:=m0609 \
  host:=192.168.1.100 \
  port:=12345
이 단계는 연결과 상태 확인 목적이다. 좌표 기록 전까지 자동 동작 명령을 보내지 않는다.

3. base_link 기준 link_6 좌표 읽기

터미널 2에서 실행한다.

cd /home/ssu/Azas
source /opt/ros/humble/setup.bash
source /home/ssu/Azas/install/local_setup.bash

ros2 run tf2_ros tf2_echo base_link link_6

예시 출력:

Translation: [0.512, -0.083, 0.274]
Rotation: in Quaternion [0.000, 0.707, 0.000, 0.707]
값 의미
Translation [x, y, z] base_link 기준 손목 끝 위치, 단위 meter
Rotation Quaternion [x, y, z, w] link_6의 방향

4. 좌표 찍는 절차

  1. 로봇을 HOME 자세에 둔다. 교안 기준은 J1=0, J2=0, J3=90, J4=0, J5=90, J6=0.
  2. 티치펜던트 또는 직접교시 모드로 로봇 끝단을 원하는 위치에 가져간다.
  3. 터미널 2의 tf2_echo에서 Translation과 Rotation을 기록한다.
  4. 다음 위치로 이동해서 같은 방식으로 반복한다.

고정 좌표로 기록할 위치 예시:

dispenser_1_front
dispenser_2_front
dispenser_3_front
dispenser_4_front
floor_place
컵 side grip은 고정 좌표로 기록하지 않는다. 컵 위치는 비전 파이프라인이 매번 /jarvis/tumbler_dispenser/tumbler_pose로 제공한다. 여기서 기록하는 것은 디스펜서, floor, collision box처럼 작업 공간에 고정된 기준점이다.

기록 형식 예시:

dispenser_2_front:
  frame_id: base_link
  position:
    x: 0.580
    y: 0.020
    z: 0.200
  orientation:
    x: 0.000
    y: 0.707
    z: 0.000
    w: 0.707

5. Collision Geometry 측정

교안 27장의 Planning Scene 방식대로 collision object는 base_link 기준의 primitive box/cylinder로 넣는다.

디스펜서 box 측정

디스펜서 하나당 아래 지점에 link_6를 가져가서 TF를 기록한다.

dispenser_1_left
dispenser_1_right
dispenser_1_front
dispenser_1_back
dispenser_1_top
dispenser_1_bottom_or_table_top

계산:

size_x = abs(front_x - back_x) + 0.02~0.03
size_y = abs(left_y - right_y) + 0.02~0.03
size_z = abs(top_z - bottom_z) + 0.02~0.03

center_x = (front_x + back_x) / 2
center_y = (left_y + right_y) / 2
center_z = (top_z + bottom_z) / 2

MoveIt collision object 형식

header.frame_id = "base_link"
primitive = SolidPrimitive.BOX
dimensions = [size_x, size_y, size_z]
primitive_pose.position = [center_x, center_y, center_z]
primitive_pose.orientation.w = 1.0
operation = ADD
디스펜서 OBJ 모델은 RViz 시각화용이고, 실제 collision에는 보수적인 box를 쓰는 것이 안전하다.

6. 컵을 잡은 상태의 Collision

컵은 월드에 고정된 박스가 아니라, 잡은 뒤에는 link_6에 붙은 물체로 다룬다.

AttachedCollisionObject
link_name = "link_6"
shape = cylinder 또는 box
offset = link_6 기준 컵 중심 위치

side grip 상태에서는 컵 중심이 link_6와 다르다. 그리퍼 길이와 컵을 문 위치만큼 offset이 필요하다.

따라서 기록해야 하는 것은 cup_side_grip의 고정 위치가 아니라 아래 값이다.

side_grip_reference:
  meaning: 컵을 side grip 했을 때 link_6 기준 컵 중심 오프셋과 컵 자세
  cup_center_offset_from_link_6:
    x:
    y:
    z:
  cup_collision:
    radius:
    height:
  preferred_gripper_orientation:
    x:
    y:
    z:
    w:

실제 컵 위치는 비전이 주는 컵 pose를 사용하고, 위 오프셋은 "잡은 뒤 컵 collision을 link_6에 어떻게 붙일지"에만 사용한다.
